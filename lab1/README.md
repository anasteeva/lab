## Kонтрольные вопросы
* Чем отличается var от let и const?
  * Областью видимости и влиянием области видимости
`var` - устаревший способ обьявления переменных. В отличие от let и const, которые имеют блочную область видимости, где переменные доступны только внутри блока в котором они были обьявлены, var имеет функциолнальную область видимости. Не имеет значения, где я объявила  переменную var, она все равно вылезет наружу и будет видна во всей функции. Из-за этого переменная, объявленная через var внутри блока, может даже перезаписать значение переменной снаружи. 
Переменная, объявленная через let и const, "живут" только внутри своего блока, поэтому за его границами она не существует и не влияют на другие переменные.

Главное отличие const, в том, что значение const нельзя изменить после объявления.
  
  * Поднятием переменных. 
 в js обьявление перменных обрабатываются раньше, чем выполняется сам код. JS сначала просматривает код и сразу регистрирует переменные в памяти. Только после этого он начинает выполнять код сверху вниз (hoisting). В случае var, если обьявить переменную после команды выводящию данные, ошибки не будет, так как она уже считается созданной, но ее значение будет undefined. 
 С let и const все строже: переменные тоже поднимаются, но использовать их до объявления нельзя.

  * Повторным объявлением переменных
var можно объявлять несколько раз подряд, и значение просто поменяется. Ошибки при этом не будет.
let и const так делать не позволяют. Их переменную можно объявить только один раз, иначе сразу будет ошибка.

  * const нельзя объявить “пустой" и используется для значений, которые не должны изменяться.

* Что такое неявное преобразование типов в JavaScript?
  * Неявное преобразование типов - это когда JavaScript сам автоматически меняет тип данных, чтобы выполнить операцию. То есть разработчик этого напрямую не указывает, язык делает это за него.

  * Например, если сложить строку и число "5" + 2, JavaScript преобразует число в строку и получится "52". А если использовать оператор -, например "5" - 2, то строка наоборот станет числом и результат будет 3.

* Как работает оператор == в сравнении с ===?
  * == - это обычное сравнение. Перед проверкой JavaScript может автоматически преобразовать типы данных.

- 5 == "5" (true), потому что строка превращается в число.

  * === — это строгое сравнение. Он проверяет и значение, и тип. Никаких преобразований не делает

- 5 === "5" (false), так как число и строка это разные типы

Чаще используют ===, потому что он точнее и снижает вероятность ошибок.
