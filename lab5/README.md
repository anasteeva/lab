# Лабораторная работа №5. Работа с DOM-деревом и событиями в JavaScript

## Инструкции по запуску проекта

1. Открыть папку проекта в VS Code.
2. Открыть файл `index.html`.
3. Запустить его через **Live Server**.

Так как в проекте используются модули JavaScript, файл лучше запускать через Live Server, а не просто двойным кликом.

## Описание лабораторной работы

В этой лабораторной работе нужно было создать веб-приложение для учёта личных финансов. Можно добавлять транзакции (доходы и расходы), удалять их, просматривать подробности и следить за общим балансом.

Цель работы: 
научиться работать с DOM-деревом, событиями, формами и ES-модулями в JavaScript.

## Структура проекта

- `index.html` - главная страница
- `style.css` - стили
- `README.md` - отчёт
- `src/index.js` - главный файл
- `src/transactions.js` - работа с массивом транзакций
- `src/ui.js` - работа с DOM
- `src/utils.js` - вспомогательные функции

## Краткая документация

### `utils.js`

Файл содержит вспомогательные функции:

- `generateId()` - создаёт уникальный ID для транзакции.
- `formatDate(dateString)` - форматирует дату и время в понятный вид.
- `getShortDescription(description)` - возвращает первые 4 слова из описания.

---

### `transactions.js`

Файл отвечает за работу с массивом транзакций.

- `transactions` - массив, в котором хранятся все транзакции.
- `addTransaction(amount, category, description)` - создаёт новую транзакцию и добавляет её в массив.
- `removeTransaction(id)` - удаляет транзакцию по ID.
- `calculateTotal()` - считает общую сумму всех транзакций.

---

### `ui.js`

Файл отвечает за отображение данных на странице.

- `renderRow(transaction)` - добавляет новую строку в таблицу.
- `removeRow(id)` - удаляет строку из таблицы.
- `renderTotal(total)` - обновляет общий баланс.
- `renderDetails(transaction)` - показывает полное описание транзакции.
- `showError(message)` - показывает ошибку формы.
- `clearError()` - очищает сообщение об ошибке.

---

### `index.js`

Главный файл проекта. Он связывает остальные модули между собой.

В нём находятся обработчики событий:

- `handleFormSubmit(event)` - обрабатывает отправку формы, проверяет данные и добавляет транзакцию.
- `handleTableClick(event)` - обрабатывает клики по таблице. Через него работает удаление транзакции и просмотр подробностей.

---

## Примеры использования

### Добавление транзакции

Заполняем форму и нажимаем кнопку **Добавить транзакцию**.

Пример данных:

- сумма: `1500`
- категория: `Зарплата`
- описание: `Получена зарплата за апрель`

После добавления в таблице появляется зелёная строка, а общий баланс становится:

```text
Общий баланс: 1500.00
```

---

### Добавление расхода

Пример данных:

- сумма: `-300`
- категория: `Продукты`
- описание: `Поход в магазин за продуктами на неделю`

После добавления строка будет красной, а баланс изменится:

```text
Общий баланс: 1200.00
```

---

### Валидация формы

Если отправить форму с ошибкой, появится одно из сообщений:

```text
Введите сумму транзакции
Выберите категорию
Введите описание транзакции
Сумма должна быть числом
```

---

### Просмотр подробностей

При клике на строку таблицы под таблицей появляется полное описание транзакции.

Пример:

```text
Подробности транзакции:
ID: 17167893421234
Дата: 14.05.2025 10:30
Категория: Зарплата
Сумма: 1500
Описание: Получена зарплата за апрель
```

ID и дата могут отличаться, потому что они создаются автоматически.

---

### Удаление транзакции

При клике на кнопку **Удалить** строка пропадает из таблицы, а общий баланс пересчитывается.

## Ответы на контрольные вопросы

### 1. Каким образом можно получить доступ к элементу на веб-странице с помощью JavaScript?

Получить доступ к элементу можно через методы объекта `document`.

Например, если у элемента есть `id`, можно использовать:

```javascript
document.getElementById("id")
```

---

### 2. Что такое делегирование событий и как оно используется для эффективного управления событиями на элементах DOM?

Делегирование событий - это когда обработчик ставится не на каждый элемент отдельно, а на общий родительский элемент.

В моей работе обработчик клика стоит на всей таблице. Когда пользователь нажимает на кнопку Удалить или на строку таблицы, программа проверяет, куда именно был сделан клик.

Это удобно, потому что строки таблицы создаются через JavaScript уже после загрузки страницы. Благодаря делегированию не нужно отдельно добавлять обработчик для каждой новой кнопки.


---

### 3. Как можно изменить содержимое элемента DOM с помощью JavaScript после его выборки?

После того как элемент выбран, его содержимое можно изменить через свойства `textContent` или `innerHTML`.

Например, textContent используется для обычного текста:
```js
totalEl.textContent = "Общий баланс: 1500.00";
```

А innerHTML можно использовать, если нужно вставить HTML-разметку, например несколько строк с подробностями транзакции.

Также можно менять стили элемента через `style`, например цвет текста.

---

### 4. Как можно добавить новый элемент в DOM дерево с помощью JavaScript?

Новый элемент можно создать с помощью `document.createElement()`.

Например, в моей работе для новой транзакции создаётся строка таблицы:

```js
const row = document.createElement("tr");
```
