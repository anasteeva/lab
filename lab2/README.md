## Описание лабораторной работы

В этой лабораторной работе нужно было реализовать аналоги встроенных методов массивов JavaScript вручную, без использования самих встроенных функций (map, filter, forEach и т.д.)

Все функции написаны с использованием цикла `for`, свойства `length` и обращения к элементам массива по индексу.

## Краткая документация к проекту
### printArray(array)

Выводит элементы массива в формате:

```
Element 0: value x
```

---

### printArray1(array)

Выводит элементы массива в кратком формате:

```
0:  x
```

---

### forEach(array, callback)

Выполняет переданную функцию `callback` для каждого элемента массива

Колбэк вызывается с аргументами:

```
callback(element, index, array)
```

Функция ничего не возвращает

---

### map(array, callback)

Создает новый массив, содержащий результаты обработки каждого элемента исходного массива

---

### filter(array, callback)

Создает новый массив только из тех элементов, для которых `callback` возвращает `true`

---

### find(array, callback)

Возвращает первый элемент массива, удовлетворяющий условию  
Если подходящий элемент не найден, возвращает `undefined`

---

### some(array, callback)

Проверяет, существует ли хотя бы один элемент массива, удовлетворяющий условию
Возвращает `true` или `false`

---

### every(array, callback)

Проверяет, удовлетворяют ли все элементы массива условию
Если хотя бы один элемент не подходит, возвращает `false`

---

### reduce(array, callback, initialValue)

Последовательно обрабатывает элементы массива и накапливает результат в аккумуляторе

Используется для получения одного итогового значения, например суммы элементов массива

---

## Примеры использования

### forEach

```js
// Проходит по массиву и выполняет действие для каждого элемента
forEach([1, 2, 3], function(element, index) {
    console.log("Element " + index + ": " + element);
});
```

---

### map

```js
// Создает новый массив, где каждый элемент умножен на 2
let doubled = map([1, 2, 3], function(x) {
    return x * 2;
});

console.log(doubled); // [2, 4, 6]
```

---

### filter

```js
// Оставляет только четные числа
let even = filter([1, 2, 3, 4], function(x) {
    return x % 2 === 0;
});

console.log(even); // [2, 4]
```

---

### find

```js
// Находит первое четное число в массиве
let firstEven = find([1, 3, 5, 6], function(x) {
    return x % 2 === 0;
});

console.log(firstEven); // 6
```

---

### some

```js
// Проверяет, есть ли хотя бы одно четное число
let hasEven = some([1, 3, 5], function(x) {
    return x % 2 === 0;
});

console.log(hasEven); // false
```

---

### every

```js
// Проверяет, все ли числа четные
let allEven = every([2, 4, 6], function(x) {
    return x % 2 === 0;
});

console.log(allEven); // true
```

---

### reduce

```js
// Считает сумму всех чисел массива
let sum = reduce([1, 2, 3], function(acc, x) {
    return acc + x;
}, 0);

console.log(sum); // 6
```

## Ответы на контрольные вопросы

### В чем преимущества использования колбэков при работе с массивами?

Колбэки делают функции универсальными. Одна и та же функция может выполнять разные действия в зависимости от переданной логики.

Например, функция `map` всегда проходит по массиву, но результат зависит от того, какую операцию выполняет колбэк.

Также колбэки уменьшают дублирование кода - не нужно каждый раз писать новый цикл.

---

### Какие проблемы могут возникать при использовании колбэков и как их избежать?

Основные сложности:

- сложно понять, какие аргументы получает колбэк
- можно перепутать порядок аргументов
- забыть вернуть значение внутри колбэка
- код может стать менее читаемым при большом количестве вложенных функций

Избежать проблем можно некоторыми способами:

- внимательно следить за сигнатурой функции
- использовать понятные имена параметров
- добавлять комментарии
- тестировать функции на простых примерах

---

### Как реализовать функции map, filter, find, some, every и reduce без использования встроенных методов массивов?

Все эти функции можно реализовать вручную с помощью цикла `for`

Основные принципы:

- проход по массиву слева направо
- обращение к элементам по индексу
- использование условий (`if`)
- формирование нового массива через `push` (для map и filter)
- возврат результата через `return`

:

- map - преобразует каждый элемент и сохраняет в новый массив
- filter - добавляет в новый массив только подходящие элементы
- find - возвращает первый подходящий элемент
- some - возвращает true, если найден хотя бы один подходящий элемент
- every - возвращает false при первом неподходящем элементе
- reduce - накапливает результат в аккумуляторе
